Mensajería Concurrente en Go. Este es un sistema de mensajería en tiempo real diseñado para ser rápido, ligero y capaz de manejar a muchos usuarios al mismo tiempo sin despeinarse. El Equipo: Juan Andino, Marco Pino, Matías Reyes. ¿De qué trata este proyecto? Queríamos construir algo que funcionara de verdad bajo presión. Este servidor de chat utiliza WebSockets para que los mensajes lleguen al instante (baja latencia) y aprovecha las Goroutines de Go para que cada conexión sea independiente y eficiente.Características Principales. Tiempo Real: Comunicación bidireccional mediante WebSockets (usando Gorilla). Arquitectura Hub & Client: Un centro de control (Hub) organiza quién entra, quién sale y reparte los mensajes. Alta Concurrencia: Gracias a Go, podemos manejar miles de clientes simultáneos con un consumo mínimo de memoria. Persistencia Asíncrona: Los mensajes se guardan en la base de datos sin detener la fluidez del chat. Stack Tecnológico. Elegimos herramientas modernas y potentes de lenguaje Go por su manejo nativo de la concurrencia. Framework Web: Gin Gonic para una API rápida. Base de Datos: GORM (ORM para Go) para manejar la persistencia de forma sencilla. Seguridad: Autenticación mediante JWT (JSON Web Tokens). Arquitectura del SistemaEl corazón del proyecto es un servidor central que hace de intermediario entre todos los usuarios. Autenticación: El usuario se loguea y recibe un token JWT.Conexión: Se abre un "puente" (WebSocket) hacia nuestro servidor.El Hub: Esta estructura registra al usuario en una sala y se encarga de que sus mensajes lleguen a los demás.Guardado Inteligente: Mientras tú chateas, el servidor guarda los mensajes en segundo plano para que no pierdas el historial. Próximos Pasos. Actualmente estamos en la fase de Prototipo Hub & Client. Lo que viene es: Finalizar la integración completa con la base de datos relacional. Refinar la seguridad y límites de velocidad (Rate Limiting). Desplegar el sistema en un servidor virtual para que pueda usarse desde cualquier lugar.
